# 银行审计系统深度重构技术总结 (V3.0 - V7.0)

## 1. 核心愿景：从“模板匹配”到“语义审计”
早期的系统高度依赖于固定的 Word 模板映射。在本次迭代中，我们彻底转向了**通用语义提取**。系统不再关心表格的具体坐标，而是通过 LLM 识别“谁在做什么业务，拥有什么权限”。

## 2. 关键架构组件

### A. 智能管线渲染 (The Pipeline)
- **并发加速**: 引入 `ThreadPoolExecutor` 并发处理独立的 OCR 证件任务，使多附件场景耗时缩减 40%+。
- **串行深度分析**: 保持文档对比与全局汇总的串行执行，确保 LLM 拥有完整的业务上下文。

### B. 三级风险核查体系
1. **硬核比对 (Hard Comparison)**：代码逻辑层执行，针对账号、人名、证件号进行 100% 精确匹配。
2. **逻辑语义分析 (Semantic Analyzer)**：LLM 扫描，识别业务动作（开通 vs 注销）及权限超配（多选了 EFlow 之外的限额或功能）。
3. **全局智脑 (Global Aggregator)**：对多份文档（Word、PDF、OCR、EFlow）进行交叉验证，识别身份假冒或跨文档冲突。

### C. 视觉重塑：Saas Cloud 风格
- **配色**: 采用深蓝色 (#001529) 侧重专业稳定性，辅以灰白背景增加空灵感。
- **交互**: 规范化 Button、Card 和 Modal 组件，具备标准的 B 端管理系统体验。

## 3. 逻辑挑战与演进 (护照解析案例)
针对护照解析中常见的“P 型干扰”（类型标识导致姓名错误），我们实施了：
- **Prompt 硬引导**: 强制要求 LLM 优先解析底部的国际标准机器码 (MRZ)。
- **逻辑熔断**: 若代码层发现名字为单字母，则强制拒绝结果并触发高精度再次解析。

## 4. 下一步演化方向
- **Vision 模型接入**: 针对复杂的证件图片，直接利用 Vision 能力替代传统 OCR + 语义的双跳。
- **高并发调优**: 针对企业级部署增加 Rate Limiting 与 Token 计数器。
